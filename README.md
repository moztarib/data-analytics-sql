## Project Overview

**Overview**: Using  SQL, we query large scale sales transactions data of 51,290 rows (originally csv) from a global retail business spanning 4 years and 13 regions worldwide. We do this by connecting SQLite DBMS within a Python environment.

**Objective**: To identify which specific products, regions, and customer segments are generating profit and which are quietly losing money despite high sales.

## Database Schema Design

We normalised our raw CSV into a star schema with 1 fact table and 3 dimension tables using pandas. Like the following example: 
```jupyter notebook
dim_customers = df[['customer_name', 'segment']].drop_duplicates().reset_index(drop = True)
```

- Similarly for dim_locations, dim_products, and fact_orders table:

A schematic sketch behind the table schema design:
```bash

dim_customers 
                |----→ fact_orders ←-----| dim_products
dim_location

```
| Table | Columns |
|---|---|
| `dim_customers` | customer_name (pk), segment |
| `dim_products` | product_id (pk), product_name, category, sub_category |
| `dim_locations` | location_id (pk), region, market, country, state |
| `fact_orders` | order_id, order_date, ship_date, ship_mode, sales, quantity, discount, profit, shipping_cost, order_priority, year, customer_name (FK), product_id (FK), location_id (FK) |

## Some of our main FINDINGS from each section:
---
### Advanced Analytics (CTE, JOIN, RANK(), GROUP BY)
- Tables is the only sub-category under loss. It is losing ~$64,000 in profit, DESPITE generating a sizeable $289,686 in sales revenue!
- The Year 2013 came out as the strongest growth year with 32.89% Year-on-Year profit growth, and then suffered a hit reaching back to the levels similar to 2012, in 2014.
- Southeast Asia has the weakest profit margin DESPITE a $532,172 in revenue.
---
### Segmentation: CASE WHEN
- Only 41 high performing products drive the majority of profit. A total of 674 products actively lose money which is ~ 7% of total product catalogue.
- 73 VIP (>10,000 in total sales) and 399 Loyal customers (>5000 in total sales) make up virtually the entire customer base! \
- We identified Zero occasional customers (<1000 in total sales) which is also suggesting a B2B focused business model.
---
### Time Series: STRFTIME()
- Nov, Dec dominate the revenue generation as holiday months.
- Feb and Jan are weakest which is typical post-holiday retail slowdown.
---

### Main conclusions:
> Heavy discounting on furniture might be a cause of margin destruction on Tables. All other 16 sub-categories are profitable.

> The business strategy in Southeast Asia needs a review.

> If 674 producst are losing money, the business should review pricing (or discontinue some products?)

> Retaining top customers is critical. Losing one VIP will affect business significantly.

> Business should focus on inventory reconciliation and strategy building during Feb and Jan months.

## How to Run

1. Clone this repository
2. Download the dataset from the Kaggle link above and place it in `datasets/`
3. Install required libraries:
```
   pip install pandas sqlalchemy ipython-sql prettytable jupyter
```
4. Open and run `notebooks/01_load_data.ipynb` — this establishes the connection and builds the database 
5. Open any `.sql` file in `scripts/` and copy queries into the notebook 
   using `%%sql` to see results
